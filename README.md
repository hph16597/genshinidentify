# 原神角色头像使用率识别工具

这是一个本地运行的网页工具。它会读取使用率截图，自动寻找“头像在上、使用率在下”的角色卡片，识别角色中文名和使用率，并导出 Excel。

当前重点适配两类截图：

- “提瓦特小助手”深渊使用率统计：带 S+、S、A、B、C 分段。
- 幽境危战统计：蓝色背景、小卡片密集排列，可能带 NEW 或涨跌小角标。

## 启动工具

双击项目根目录中的 `run.bat`。

第一次启动会自动创建本地 Python 环境并安装组件，可能需要几分钟。以后再次使用仍然双击 `run.bat`。

如果浏览器没有自动打开，请在浏览器地址栏输入：

`http://localhost:8501`

不要关闭黑色命令窗口；关闭它会停止工具。

## 识别截图

1. 在网页中点击“选择一张截图”并上传。
2. 程序自动寻找头像区域，并在原图上显示绿色框。
3. 检查绿色框是否完整覆盖头像。
4. 如有误检或漏检，在表格中修改、删除或新增头像框。
5. 点击“开始识别并导出结果”。
6. 下载 Excel 结果。

## Excel 输出格式

导出的 `output/result.xlsx` 固定只有 4 列：

| 列 | 字段 | 示例 |
|---|---|---|
| 第一列 | 排名 | 1 |
| 第二列 | 角色中文名 | 芙宁娜 |
| 第三列 | 使用率 | 97.9 |
| 第四列 | 使用率文本 | 97.9% |

排序规则：

- 按使用率数字从高到低排序。
- 相同使用率按原图从上到下、从左到右的顺序排序。
- 使用率识别失败的行排在最后。

网页中仍会显示置信度、备选角色、头像框坐标和是否需要复核；这些调试字段不会写入最终 Excel。

## 新角色头像

识别角色需要本地参考头像。标准头像放在项目根目录的 `avatars` 文件夹中。

- 文件名必须是角色中文名，例如 `玛薇卡.webp`、`芙宁娜.webp`。
- 支持 PNG、JPG、JPEG、WEBP、BMP。
- 新角色可以通过网页左侧“联网更新新角色头像”按钮补充。
- 如果联网更新失败，也可以手动把头像图片放入 `avatars` 文件夹。

联网更新会尽量从 `genshin-db-api` 获取角色资料，并在必要时使用 Enka 静态图片地址作为备用来源。

## 手动修正头像框

截图版式可能变化，所以网页保留手动修正能力：

- 调整框：修改表格中的 `X`、`Y`、`宽`、`高`。
- 删除误检框：删除表格中对应行。
- 补充漏检框：点击表格底部的 `+`，填写坐标和大小。
- 可视化补框：展开“可视化补充一个头像框”，用滑块移动蓝色框后添加。

修改后点击“应用表格修改并刷新预览”。

## 如何测试

使用项目自带环境运行：

```powershell
.\.venv\Scripts\python.exe tests\test_usage_rate_workflow.py
```

测试内容包括：

- 使用率文本标准化。
- Excel 四列输出和排序。
- 三张样例图的头像区域检测。
- 幽境危战样例不把顶部 UI 当头像。

也可以手动测试：

1. 双击 `run.bat`。
2. 分别上传三张样例图。
3. 确认绿色框覆盖头像。
4. 导出 Excel。
5. 检查 `output/result.xlsx` 是否只有 4 列，并按使用率降序排列。

## 如果失败如何回滚

如果本次改造后不好用，可以恢复或删除这些文件：

- 恢复：`detector.py`、`usage_ocr.py`、`matcher.py`、`app.py`、`README.md`。
- 删除：`result_export.py`、`avatar_updater.py`、`tests/`。
- 删除生成结果：`output/result.xlsx`、`output/result.csv`、`output/crops/`。

`avatars` 原有头像不要删除。如果联网更新下载了错误头像，只删除对应新增图片即可。
